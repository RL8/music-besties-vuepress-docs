# Dashboard View

The Dashboard View is the main entry point of the Music Besties application v1.1. It displays a grid of era cards with enhanced sorting options and metrics, providing users with a comprehensive overview of their music preferences.

# Screen Overview

Purpose: Provide a visual overview of all music eras with enhanced information display and improved organization

Key Enhancements in v1.1:

  • Redesigned card layout with All Eras card taking the full first row
  • Enhanced information display with comprehensive metrics
  • Improved visual hierarchy with dedicated spaces for different content types
  • Integration of bestie matching information into cards
  • Single sharing access point through a dashboard "Share" button

# Card Layout and Design

# Card Positioning

  • All Eras card takes full first row for prominence
  • Individual era cards maintain current positioning and adjust without Eras card
  • Responsive grid layout adapts to different screen sizes

# Card Design

  • Top 2/3 of card reserved for name and emoji to have full prominence
  • Lower 1/3 for summarized info regarding card
  • Top left corner preserved for number denoting rank
  • Tapping anywhere on card leads to right side bar
  • Era-specific styling applied through the theme system

# Card Content

  • For All Eras card:

    • Row of era-specific emojis representing each era in order of release
    • Current rank for each era (blank if unranked)
    • Top bestie match information

  • For individual era cards:

    • Space shared equally by content
    • Chips with emoji and relevant number
    • Chips for: Era ranking, Songs ranked (e.g., 3/22), number of comments by the user, number of reviews/comments for this era by all app users
    • Bestie match information can be incorporated into cards

# Component Details

# EraCard.vue

The EraCard component displays information about a single music era, with era-specific styling applied through the theme system.

Properties:

  • era: Object containing era information (id, name, emoji, etc.)
  • rank: Number indicating the era's rank in the user's list
  • selectedCount: Number of songs selected for this era
  • totalSongs: Total number of songs in this era
  • commentCount: Number of comments by the user for this era
  • communityCount: Number of reviews/comments for this era by all users
  • bestieMatch: Object containing bestie match information (optional)

Styling:

  • Each card uses era-specific theme variables for colors and fonts
  • Cards have subtle hover and active states for better interaction feedback
  • Metrics use consistent iconography for clear visual communication
  • Rank number uses era-specific primary color

# AllErasCard.vue

New in v1.1, this component displays an overview of all eras with their emojis and ranks.

Properties:

  • eras: Array of era objects with their information
  • rankings: Object mapping era IDs to their ranks
  • topBestieMatch: Object containing information about the user's top bestie match

Styling:

  • Takes full width of the first row
  • Uses star emoji (ğŸŒŸ) as its main icon
  • Displays era emojis in a horizontal row
  • Shows rank numbers below emojis when available

# ShareButton.vue

New in v1.1, this component provides a single access point for sharing functionality.

Properties:

  • label: String for the button text
  • disabled: Boolean to control button state

Events:

  • @click: Emitted when the button is clicked

# Sharing Functionality

New in v1.1, the sharing functionality has been streamlined:

  • Removed From: Right hand side bar and any other places
  • Single Access Point: Only sharing outlet is a button on the dashboard labeled "Share"
  • Destination: Takes user to external share page, accessible via unique link
  • Preview Mode: Allows final adjustments (profile photo, username, bio, show/hide content)
  • Analytics: Tracks number of visits to link and stores in user profile

# Responsive Design

The Dashboard View implements responsive design principles:

  • Mobile: 2-column grid with All Eras card taking full width
  • Tablet: 3-column grid with All Eras card taking full width
  • Desktop: 3-column or more grid with larger cards and additional information

# Accessibility Features

  • All interactive elements have appropriate ARIA labels
  • Color contrast meets WCAG 2.1 AA standards
  • Keyboard navigation is fully supported
  • Screen reader announcements for dynamic content changes
  • Focus management for modal dialogs
