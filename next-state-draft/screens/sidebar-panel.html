# Sidebar Panel

The Sidebar Panel is a context-sensitive panel that slides in from the right side of the screen in the Music Besties application v1.1. It provides detailed information and interaction options for music eras, songs, community content, and social connections.

# Screen Overview

Purpose: Provide a detailed view of user content, community interactions, and social connections with an intuitive navigation structure

Key Enhancements in v1.1:

  • Implemented three-layer navigation approach for clearer content organization
  • Reorganized tabs into "Your Journal | Community | Besties" with no nested tabs
  • Shifted focus from era-level to song-level reviews and comments
  • Added new Besties tab for social connections
  • Improved visual hierarchy with consistent styling across all content types

# Navigation Structure

The Sidebar Panel implements a three-layer navigation approach:

# Layer 1: Main Tabs

  • Your Journal: Personal content including rankings, reviews, and notes
  • Community: Content from other users, including reviews and comments
  • Besties: Matching and connections with users who have similar taste

# Layer 2: Content Lists

  • Era List: When viewing All Eras in Your Journal tab
  • Song List: When viewing a specific era in Your Journal tab
  • Review List: When viewing community content
  • Bestie List: When viewing besties tab

# Layer 3: Detail Views

  • Era Detail: Your review and comments for an era
  • Song Detail: Your review and comments for a song
  • Review Detail: A specific community review with comments
  • Bestie Detail: Detailed view of a specific bestie match

# Component Details

# SidebarHeader.vue

The header component with navigation breadcrumbs and close button.

Properties:

  • breadcrumbs (Array): Array of breadcrumb objects with name and ID
  • showClose (Boolean): Whether to show the close button

Events:

  • @close: Emitted when the close button is clicked
  • @navigate: Emitted when a breadcrumb is clicked

# TabNavigation.vue

The tab navigation component for switching between main tabs.

Properties:

  • tabs (Array): List of tab objects with id, title, and active properties
  • activeTabId (String): ID of the currently active tab

Events:

  • @tabChange: Emitted when a tab is clicked, with the new tab ID

# ContentList.vue

Component for displaying lists of content items.

Properties:

  • items (Array): List of content items with appropriate properties
  • type (String): Type of list (eras, songs, reviews, besties)
  • actions (Array): Available actions for list items

Events:

  • @select: Emitted when an item is selected
  • @action: Emitted when an action is triggered on an item

# DetailView.vue

Component for displaying detailed content views.

Properties:

  • content (Object): Content object with all necessary details
  • type (String): Type of detail view (era, song, review, bestie)
  • actions (Array): Available actions for the content

Events:

  • @action: Emitted when an action is triggered on the content

# User Interactions

Interaction Result Implementation
Tap tab Switches between main tabs Emits @tabChange event with tab ID
Tap breadcrumb Navigates to previous level Emits @navigate event with breadcrumb ID
Tap close button Closes sidebar panel Emits @close event
Tap list item Opens detail view for that item Emits @select event with item ID
Tap action button Performs the associated action Emits @action event with action type
Tap like button Toggles like status on review/comment Updates like count and status
Tap connect button Initiates connection with user Sends connection request

# Theme Integration

The Sidebar Panel components use era-specific theming to maintain visual consistency:

  • Header background uses a light version of the era's primary color
  • Era name uses the era's font family
  • Tab indicators use the era's primary color
  • Rating stars use the era's secondary color

# Accessibility Considerations

  • All interactive elements have appropriate ARIA labels
  • Color contrast meets WCAG 2.1 AA standards
  • Keyboard navigation is fully supported
  • Screen reader announcements for tab changes and content updates
  • Focus management for nested navigation
