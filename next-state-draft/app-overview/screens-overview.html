# Screens Overview

The Music Besties application v1.1 consists of multiple screens and views that provide a comprehensive user experience for organizing, reviewing, and sharing music preferences. This section provides detailed documentation for each screen, including its components, interactions, and visual layout.

# Primary Screens

# Dashboard View

The Dashboard View is the main entry point of the Music Besties application. It displays a grid of era cards with enhanced sorting options and metrics.

Key Enhancements in v1.1:

  • Card Positioning: All Eras card takes full first row, with individual era cards below
  • Card Design: Top 2/3 of card reserved for name and emoji, lower 1/3 for summarized info
  • Card Content: Enhanced information display with chips showing era ranking, songs ranked, comment counts, and community activity
  • Bestie Integration: Cards can show bestie match information, with the All Eras card displaying the top bestie match

# Edit Rankings View

The Edit Rankings View provides an interface for users to select and rank items within a specific music era.

Key Enhancements in v1.1:

  • Quick Rank Feature: New option to rank just top 3 eras and top 3 songs per era
  • Selection Modal: When clicking "Edit Rankings," users can choose between quick rank and full ranking
  • Enhanced Sorting: Improved sorting options that affect all tabs and save user preferences
  • History Integration: Better tracking of ranking changes over time

# Sidebar Panel

The Sidebar Panel is a context-sensitive panel that slides in from the right side of the screen, providing detailed information and interaction options.

Key Enhancements in v1.1:

  • Three-Layer Navigation: Clear structure with main tabs, content lists, and detail views
  • Tab Organization: "Your Journal | Community | Besties" tabs with no nested tabs
  • Content Focus: Shifted from era-level to song-level reviews and comments
  • Bestie Integration: New tab for connecting with users who have similar taste

Navigation Structure:

  1. Layer 1: Main Tabs

    • Your Journal: Personal content
    • Community: Content from other users
    • Besties: Matching and connections

  2. Layer 2: Content Lists

    • Era List: When viewing All Eras
    • Song List: When viewing a specific era
    • Review List: When viewing community content
    • Bestie List: When viewing besties tab

  3. Layer 3: Detail Views

    • Era Detail: Your review and comments for an era
    • Song Detail: Your review and comments for a song
    • Review Detail: A specific community review with comments

# New Screens in v1.1

# Public Profile View

The Public Profile View allows users to share their rankings and reviews through a unique link.

Key Features:

  • Customizable Content: Users can adjust what information is visible
  • Preview Mode: See how the profile appears to others before sharing
  • Visitor Limits: Non-logged-in visitors see only top 3x3 content (top 3 eras, top 3 songs per era)
  • Comparison Encouragement: Visitors are prompted to create their own account to compare preferences

# Bestie Matching View

The Bestie Matching View displays a league table of users with similar music taste.

Key Features:

  • Matching Algorithm: Based on top 3 eras and top 3 songs per era
  • Connected vs. Discover: Clear separation between existing connections and potential matches
  • Era-Specific Matching: Different scores for overall similarity and era-specific similarity
  • Verification Integration: Option to show only verified users

# Support Pages

Enhanced help and information pages with improved functionality.

Key Enhancements:

  • Autosave: Changes are automatically saved with subtle alert confirmation
  • Modal Interface: No need for drawer panel, modal is adequate
  • Navigation Accessibility: Left sidebar remains accessible while modal is open
  • Visual Feedback: Currently open page is highlighted for clarity

# User Interaction Flows

# Sharing Flow

  1. User taps "Share" button on dashboard
  2. User is taken to external share page
  3. User can make final adjustments (profile photo, username, bio, show/hide content)
  4. User shares unique link
  5. System tracks visits to the link

# Connection Flow

  1. Outgoing: User initiates connection with another user via their profile page
  2. Incoming: Another user initiates connection with current user
  3. Connected users have full access to profile (excluding marked private content)
  4. Connection dates are recorded and displayed

# Privacy Control Flow

  1. User is informed that info is publicly visible to other app users
  2. User can mark comments or reviews as private for permitted users only
  3. User can see others on their league table
  4. User can opt in those who can see private content

# Headers and Footers

  • Standardized throughout the app following best practices
  • Freeze header functionality for better navigation
  • Special accommodations for each view
  • Fan app disclaimer: Removed from bottom with one-time agreement for each user
