# Data Overview

This section provides a comprehensive overview of the data management approach in the Music Besties application v1.1.

# Data Architecture

Music Besties uses a structured data model to organize music era and song information. The application implements a unidirectional data flow pattern with several primary types of data:

  1. Static Era Data: Pre-defined information about music eras including identifiers, titles, and song lists
  2. User Preference Data: User-generated content including selections, rankings, ratings, and reviews
  3. History Data: Timestamped records of ranking changes for tracking user preferences over time
  4. User Account Data: Optional user account information and authentication status
  5. Social Data: User matching data and Reddit verification status

The data structure is designed to be lightweight and efficient, with a clear separation between core era data and user-specific preferences. This separation allows the application to reset user data without affecting the underlying era information if needed.

# Storage Mechanisms

# LocalStorage

The application uses the browser's LocalStorage API to persist user data between sessions. This approach allows:

  • No server-side storage requirements for core functionality
  • Data remains available even when offline
  • Simple API for storing and retrieving JSON data

# In-memory State

During runtime, the application maintains an in-memory state that is synchronized with LocalStorage, providing:

  • Fast access to data during user interactions
  • Ability to make temporary changes before persisting
  • Simplified state management through Vue.js reactivity

# Supabase Integration

Version 1.1 introduces Supabase integration for user authentication and social features:

  • User accounts with optional guest mode
  • Secure authentication with email/password or social providers
  • Data synchronization between devices
  • User profile management
  • Privacy controls for shared content

# New Data Features in v1.1

Version 1.1 introduces several new data management features:

  1. Ranking History:

    • Timestamped records of ranking changes
    • Ability to view and restore previous rankings
    • Change tracking with detailed diff information

  2. User Accounts:

    • Optional registration with email/password
    • Guest mode for users who prefer not to create an account
    • Upgrade path from guest to registered user

  3. Social Features:

    • Reddit verification for user identity confirmation
    • Bestie matching algorithm to find users with similar taste
    • Privacy controls for shared content

  4. Enhanced Reviews:

    • Expanded review capabilities with ratings and comments
    • Review history tracking
    • Social sharing options

# Data Flow

The application follows a unidirectional data flow pattern:

  1. User interactions trigger actions
  2. Actions update the in-memory state
  3. State changes are persisted to LocalStorage and/or Supabase
  4. UI components react to state changes and update accordingly

This pattern ensures predictable application behavior and simplifies debugging by making data changes traceable.
