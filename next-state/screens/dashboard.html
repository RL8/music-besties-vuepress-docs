# Dashboard View

The Dashboard View is the main entry point of the Music Besties application v1.1. It displays a grid of era cards, each representing a music era that the user can interact with, including new history indicators for tracking ranking changes over time.

# Screen Overview

Purpose: Provide a visual overview of all music eras and their current status (ratings, selections, history)

Primary Components:

  • EraCard.vue (multiple instances in grid layout)
  • HeaderBar.vue (app title, navigation menu, edit button)
  • HistoryIndicator.vue (shows ranking history availability)
  • FooterDisclaimer.vue (legal text)

Detailed visualization of the responsive grid interface showing era cards with ratings, selection counts, history indicators, and emoji icons. The header includes a hamburger menu icon and Edit button, while the footer contains a disclaimer.

# Component Details

# EraCard.vue

The EraCard component displays information about a single music era, with era-specific styling applied through the theme system.

Properties:

  • era: Object containing era information (id, name, emoji, etc.)
  • selectedCount: Number of songs selected for this era
  • totalSongs: Total number of songs in this era
  • rating: Average user rating for this era (1-5)
  • hasHistory: Boolean indicating if ranking history exists

Styling:

  • Each card uses era-specific theme variables for colors and fonts
  • Cards have subtle hover and active states for better interaction feedback
  • Rating stars use the primary color from the era's theme
  • Selection counter uses secondary color from the era's theme

Events:

  • @click: Emitted when the card is clicked
  • @historyClick: Emitted when the history indicator is clicked
  • @longpress: Emitted when the card is long-pressed (mobile)

# HeaderBar.vue

The header bar provides navigation and editing functionality.

Properties:

  • title: String for the app title
  • showEditButton: Boolean to control edit button visibility

Events:

  • @menuClick: Emitted when the hamburger menu icon is clicked
  • @editClick: Emitted when the edit button is clicked

# HistoryIndicator.vue

New in v1.1, this component shows when ranking history is available for an era.

Properties:

  • hasHistory: Boolean indicating if history exists
  • historyCount: Number of history entries available

Events:

  • @click: Emitted when the indicator is clicked

# FooterDisclaimer.vue

Simple component displaying legal disclaimer text.

# User Interactions

Interaction Result Implementation
Click hamburger menu Opens the Left Sidebar Navigation Emits @menuClick event
Click Edit button Enters Edit Rankings View Emits @editClick event
Click on an era card Opens the Sidebar Panel with era's song list in Your Journal tab Emits card @click event with era ID
Click on history indicator Opens the Sidebar Panel with song details showing review history Emits @historyClick event with era ID and song ID
Long-press on an era card Shows additional options menu Emits card @longpress event

# State Management

The Dashboard View interacts with the following state:

  • Era data: Static information about music eras
  • User selections: Which songs are selected for each era
  • User ratings: Average ratings for each era
  • History data: Availability of ranking history for each era

# Responsive Design

The Dashboard View implements responsive design principles:

  • Mobile: 2-column grid with stacked cards
  • Tablet: 3-column grid with medium-sized cards
  • Desktop: 3-column grid with larger cards and additional information

# Accessibility Features

  • All interactive elements have appropriate ARIA labels
  • Color contrast meets WCAG 2.1 AA standards
  • Keyboard navigation is fully supported
  • Screen reader announcements for dynamic content changes

# New in v1.1

The Dashboard View has been enhanced in v1.1 with the following features:

  1. History Indicators: Visual indicators showing when ranking history is available
  2. Enhanced Era Cards: Updated design with improved visual hierarchy
  3. Rating Display: Star ratings now show the user's average rating for each era
  4. Theme Integration: Cards now use the era-specific theming system for consistent styling
  5. Long-press Actions: Additional context menu on mobile devices
