# Edit Rankings View

The Edit Rankings View provides an interface for users to select and rank items within a specific music era, with a new 24-hour cooldown period between ranking saves to encourage thoughtful, deliberate choices.

# Screen Overview

Purpose: Allow users to select favorite songs from an era and arrange them in a personalized ranking order, with a deliberate approach enforced by the cooldown period

Primary Components:

  • CooldownTimer.vue (displays time until next ranking save is available)
  • SelectionTabs.vue (tab navigation between eras)
  • ItemChip.vue (multiple instances for selectable songs)
  • HeaderControls.vue (cancel/save buttons, title)
  • SelectionCounter.vue (shows progress)

# Component Details

# CooldownTimer.vue

New in v1.1, this component displays the time remaining until the next ranking save is available.

Properties:

  • lastSaveTime: Date object representing when the last save occurred
  • cooldownHours: Number of hours required between saves (default: 24)

Methods:

  • getTimeRemaining(): Calculates time until next save is available
  • formatTimeRemaining(): Formats the remaining time in a human-readable format

Styling:

  • Uses era-specific theme variables for consistent styling
  • Changes color based on time remaining (red when locked, green when available)

# SelectionTabs.vue

The tab navigation component for switching between different eras.

Properties:

  • eras: Array of era objects
  • activeEraId: String ID of the currently active era
  • selectionCounts: Object mapping era IDs to selection counts

Events:

  • @tabChange: Emitted when a tab is clicked, with the new era ID

Styling:

  • Horizontally scrollable tab bar
  • Active tab indicator uses era-specific primary color
  • Selection counts use era-specific secondary color

# ItemChip.vue

Interactive component for selecting and ranking songs.

Properties:

  • item: Object containing song information
  • selected: Boolean indicating if the item is selected
  • rank: Number indicating the item's rank (if selected)
  • disabled: Boolean to disable interaction

Events:

  • @select: Emitted when an unselected item is clicked
  • @deselect: Emitted when a selected item is clicked

Styling:

  • Uses era-specific theme variables for colors and fonts
  • Selected items have blue background with rank number
  • Unselected items have subtle border

# HeaderControls.vue

The header component with cancel and save buttons.

Properties:

  • title: String for the screen title
  • saveEnabled: Boolean to enable/disable the save button
  • saveText: String for the save button text

Events:

  • @cancel: Emitted when the cancel button is clicked
  • @save: Emitted when the save button is clicked

# SelectionCounter.vue

The SelectionCounter component displays progress information.

Properties:

  • selectedCount: Number of items currently selected
  • totalCount: Total number of items available
  • maxSelections: Maximum number of selections allowed (optional)

Styling:

  • Uses era-specific theme variables for consistent styling
  • Progress indicator changes color based on selection status

# User Interactions

Click Cancel button Returns to Dashboard View without saving changes Emits @cancel event
Click Save button Saves selections and returns to Dashboard View Emits @save event
Click tab Switches to different era Emits @tabChange event with era ID
Click unselected item Adds item to selection Emits @select event with item ID
Click selected item Removes item from selection Emits @deselect event with item ID

# State Management

The Edit Rankings View interacts with the following state:

  • Era data: Static information about music eras and songs
  • User selections: Which songs are selected for each era
  • Ranking order: The order of selected songs
  • Last save time: When the last ranking save occurred

# Cooldown System

New in v1.1, the cooldown system enforces a 24-hour waiting period between ranking saves:

  1. When a user saves rankings, the current timestamp is stored
  2. The CooldownTimer component calculates time remaining until next save
  3. The Save button is disabled until the cooldown period has elapsed
  4. Users can still make changes to their selections during cooldown
  5. Changes are only persisted when Save is clicked after cooldown

This system encourages thoughtful, deliberate ranking decisions rather than frequent changes.

# Accessibility Features

  • All interactive elements have appropriate ARIA labels
  • Color contrast meets WCAG 2.1 AA standards
  • Keyboard navigation is fully supported
  • Screen reader announcements for selection changes

# New in v1.1

  2. Enhanced Tab Navigation: Improved visual design with era-specific styling
  3. Selection Limit Indicator: Visual feedback when approaching selection limits
  4. Theme Integration: Components now use the era-specific theming system
  5. Improved Animations: Smoother transitions when selecting/deselecting items
