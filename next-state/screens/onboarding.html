# Onboarding View

The Onboarding View is a new screen introduced in v1.1 that provides a smooth introduction to the Music Besties application for new users. It offers options for guest mode or account creation, with a special flow for users arriving via shared links.

# Screen Overview

Purpose: Introduce new users to the application and provide a frictionless entry point with guest mode, while offering a tailored experience for users arriving from shared links

Primary Components:

  • WelcomeCarousel.vue (introduction slides with app features)
  • GuestModeButton.vue (option to start using app without account)
  • RegistrationForm.vue (fields for creating new account)
  • ComparisonFlow.vue (special flow for users from shared links)
  • ProgressIndicator.vue (dots or bars showing onboarding progress)
  • SkipButton.vue (option to bypass detailed onboarding)

# Component Details

# WelcomeCarousel.vue

The carousel component that displays introduction slides with app features.

Properties:

  • slides (Array): List of slide objects with title, content, and image properties
  • currentSlide (Number): Index of the currently displayed slide
  • autoAdvance (Boolean): Whether to automatically advance slides

Events:

  • @slideChange: Emitted when the slide changes, with the new slide index
  • @complete: Emitted when the user reaches the last slide

Styling:

  • Smooth transitions between slides
  • Responsive image sizing
  • Consistent typography with application theme

# GuestModeButton.vue

Button component that allows users to start using the app without creating an account.

Properties:

  • text (String): Button text
  • subtext (String): Optional explanatory text below the button

Events:

  • @click: Emitted when the button is clicked

Styling:

  • Prominent styling to encourage quick start
  • Clear visual hierarchy compared to account creation

# RegistrationForm.vue

Form component for creating a new user account.

Properties:

  • fields (Array): List of form field objects
  • submitText (String): Text for the submit button
  • validationRules (Object): Validation rules for each field

Events:

  • @submit: Emitted when the form is submitted, with form data
  • @cancel: Emitted when the form is cancelled

# ComparisonFlow.vue

Special component for users arriving from shared links, focusing on comparison.

Properties:

  • sharedLink (String): The shared link that brought the user to the app
  • sharedUserId (String): ID of the user who shared the content
  • sharedContentId (String): ID of the specific content that was shared

Events:

  • @setupComplete: Emitted when the comparison setup is complete
  • @skip: Emitted when the user chooses to skip the comparison flow

# ProgressIndicator.vue

Visual indicator showing onboarding progress.

Properties:

  • totalSteps (Number): Total number of steps in the onboarding process
  • currentStep (Number): Current step in the onboarding process
  • style (String): Style of indicator ('dots' or 'bars')

Styling:

  • Active step highlighted with primary theme color
  • Inactive steps shown in muted color
  • Compact design to minimize space usage

# SkipButton.vue

Button allowing users to bypass detailed onboarding.

Properties:

  • text (String): Button text
  • confirmSkip (Boolean): Whether to show a confirmation dialog

Events:

  • @skip: Emitted when the skip button is clicked and confirmed

# User Flows

The Onboarding View supports multiple user flows:

# Standard Flow

  1. User opens the app for the first time
  2. Welcome screen appears with app introduction
  3. User can browse through feature highlights in the carousel
  4. User chooses to start as guest or create an account
  5. If guest mode is selected, user is taken directly to Dashboard
  6. If account creation is selected, registration form appears
  7. After successful registration, user is taken to Dashboard

# Shared Link Flow

  1. User opens a shared link from another user
  2. Special comparison onboarding appears
  3. User sees the shared content and comparison with sharer
  4. User can create an account or continue as guest
  5. User is taken to the specific shared content after onboarding

# Skip Flow

  1. User can skip detailed onboarding at any point
  2. Skip button takes user directly to Dashboard
  3. User can access account creation later through Settings

# Guest Mode

New in v1.1, guest mode allows users to:

  1. Use the app without creating an account
  2. Create and save rankings locally
  3. Write reviews and comments
  4. View community content
  5. Upgrade to a full account later (preserving their data)

Guest mode limitations:

  • Cannot connect with besties
  • Cannot verify identity
  • Data is only stored locally
  • Cannot share rankings or reviews

# Accessibility Features

  • All interactive elements have appropriate ARIA labels
  • Color contrast meets WCAG 2.1 AA standards
  • Keyboard navigation is fully supported
  • Screen reader announcements for slide changes
  • Skip links for keyboard users

# Theme Integration

The Onboarding View uses the application's theme system for consistent styling:

  • Typography follows application font hierarchy
  • Colors use theme variables for consistency
  • Component styling matches the rest of the application
  • Responsive design works across device sizes
