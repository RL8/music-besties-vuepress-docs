# Bestie Matching View

The Bestie Matching View is a new screen introduced in v1.1 that displays a league table of users with similar music taste, ranked by similarity percentage. This feature helps users connect with others who share their musical preferences.

# Screen Overview

Purpose: Display a ranked list of users with similar music taste and enable social connections between users with compatible preferences

Primary Components:

  • LeagueTable.vue (ranked list of similar users)
  • SimilarityScore.vue (visual representation of match percentage)
  • UserProfileCard.vue (user information and match details)
  • ComparisonDetails.vue (expandable section showing matching preferences)
  • FilterControls.vue (options to sort and filter matches)
  • RedditVerification.vue (account verification component)

# Component Details

# LeagueTable.vue

The main component that displays the ranked list of similar users.

Properties:

  • matches (Array): List of user match objects with similarity data
  • connected (Array): List of connected user objects
  • currentUserId (String): ID of the current user
  • viewType (String): Current view type (All Eras or specific era)
  • sortBy (String): Current sort criteria
  • verifiedOnly (Boolean): Whether to show only verified users

Events:

  • @connect: Emitted when the connect button is clicked for a user
  • @viewProfile: Emitted when the view profile button is clicked
  • @showMore: Emitted when the show more button is clicked
  • @viewChange: Emitted when view type is changed (All Eras/Specific Era)

Implementation Notes:

  • Separates connected users from potential new connections
  • Displays connection status with directional icons (↑ for outgoing, ↓ for incoming)
  • Uses era-specific emoji and colors for era chips
  • Verified users are marked with a checkmark (✓)

# SimilarityScore.vue

Visual component that displays the match percentage.

Properties:

  • score (Number): Similarity percentage (0-100)
  • size (String): Display size ('small', 'medium', 'large')
  • showLabel (Boolean): Whether to show the percentage label

Styling:

  • Color gradient based on match percentage (green for high, yellow for medium, red for low)
  • Circular progress indicator for visual representation
  • Optional text label showing exact percentage

# UserProfileCard.vue

Card component displaying user information and match details.

Properties:

  • user (Object): User information including username and verification status
  • matchPercentage (Number): Similarity percentage
  • commonFavorites (Array): List of common favorite songs or eras
  • connectionDate (Date): When the connection was established (if connected)
  • connectionDirection (String): Direction of connection ('incoming' or 'outgoing')

Events:

  • @connect: Emitted when the connect button is clicked
  • @viewProfile: Emitted when the view profile button is clicked
  • @expandDetails: Emitted when the user wants to see more details

# ComparisonDetails.vue

Expandable component showing detailed comparison between users.

Properties:

  • currentUserSelections (Object): Current user's rankings and ratings
  • matchUserSelections (Object): Match user's rankings and ratings
  • expanded (Boolean): Whether the details are expanded

Events:

  • @toggle: Emitted when the expand/collapse button is clicked

Styling:

  • Collapsible section with smooth animation
  • Side-by-side comparison of rankings
  • Highlighted matching items
  • Era-specific styling for each item

# FilterControls.vue

Component for sorting and filtering the matches.

Properties:

  • viewOptions (Array): Available view options (All Eras, specific eras)
  • sortOptions (Array): Available sort options (Match %, Recent, etc.)
  • currentView (String): Currently selected view
  • currentSort (String): Currently selected sort option
  • verifiedOnly (Boolean): Whether verified-only filter is active

Events:

  • @viewChange: Emitted when view option changes
  • @sortChange: Emitted when sort option changes
  • @filterChange: Emitted when verified-only filter changes

# RedditVerification.vue

Component for verifying user identity through Reddit.

Properties:

  • verified (Boolean): Whether the user is verified
  • redditUsername (String): Connected Reddit username
  • verificationStatus (String): Current status of verification process

Events:

  • @startVerification: Emitted when verification process is initiated
  • @checkStatus: Emitted when checking verification status
  • @disconnect: Emitted when disconnecting Reddit account

# Matching Algorithm

The Bestie Matching system uses a sophisticated algorithm to determine similarity:

  1. Top-Level Matching: Compares users' top 3 eras and their rankings
  2. Song-Level Matching: Compares specific song selections within matching eras
  3. Rating Similarity: Considers how closely ratings align for common songs
  4. Weighted Scoring: Gives higher weight to higher-ranked items
  5. Normalization: Converts raw similarity scores to percentages

The algorithm produces both:

  • Overall similarity score across all eras
  • Era-specific similarity scores for each matching era

# Reddit Verification

The Bestie Matching feature integrates with Reddit verification:

  • Only verified Reddit users appear in the league table by default
  • Verification status is indicated on user profiles
  • The system checks for subscription to positive fan communities
  • Verification helps build trust in the community

# Privacy Controls

Users have control over their visibility in the Bestie Matching system:

  • Option to hide profile from matches entirely
  • Granular control over which rankings and reviews are visible
  • Ability to block specific users from seeing their profile
  • Option to appear only to verified users

# Accessibility Features

  • All interactive elements have appropriate ARIA labels
  • Color contrast meets WCAG 2.1 AA standards
  • Keyboard navigation is fully supported
  • Screen reader announcements for dynamic content changes
  • Alternative text for era emoji and icons

# Theme Integration

The Bestie Matching components use the application's theme system:

  • Era chips use era-specific colors and fonts
  • Match percentage uses consistent color coding
  • Typography follows application font hierarchy
  • Responsive design works across device sizes
