# Theme System

This document outlines the theme system used in the Music Besties application to apply era-specific styling throughout the app.

# Overview

The theme system provides a consistent way to apply era-specific colors, fonts, and styling to various components in the application. The system is built on top of the design specifications defined in the Design Foundations section.

# Theme Variables

The application uses CSS variables to apply era-specific styling:

:root {
    --era-primary: #008080;    /* Default primary color */
    --era-secondary: #B2DFDB;  /* Default secondary color */
    --era-font: 'Outfit', sans-serif;  /* Default font */
}

These variables are updated dynamically based on the selected era.

# Default Theme

The application has a default theme that is used when no specific era is selected:

// Default app theme (Teal)
export const defaultTheme = {
  primary: '#008080',
  secondary: '#B2DFDB',
  font: "'Outfit', sans-serif"
};

# Theme Utilities

The theme system is implemented through a set of utility functions that provide access to era-specific styling.

# getEraTheme(eraNameOrId)

Retrieves the theme for a specific era based on its name or ID.

Parameters:

  • eraNameOrId (string): The name or ID of the era

Returns:

  • An object containing:
    • primary: Primary color hexcode
    • secondary: Secondary color hexcode
    • font: Font family specification

Example:

const fearlessTheme = getEraTheme('fearless');
// Returns: { primary: '#FFD700', secondary: '#F5DEB3', font: "'Arsenal', sans-serif" }

# getEraComponentStyle(eraNameOrId, componentType)

Provides component-specific styling for an era.

Parameters:

  • eraNameOrId (string): The name or ID of the era
  • componentType (string): The type of component (card, tab, chip, button, etc.)

Returns:

  • A style object with CSS properties appropriate for the component type

Component Types:

  • card: Styling for card components
  • tab: Styling for tab components
  • chip: Styling for chip components
  • button: Styling for button components

Example:

const cardStyle = getEraComponentStyle('red', 'card');
// Returns: { borderColor: '#D32F2F', backgroundColor: '#FFCDD220' }

# getEraEmoji(eraNameOrId)

Retrieves the emoji associated with a specific era.

Parameters:

  • eraNameOrId (string): The name or ID of the era

Returns:

  • The emoji character for the era

Special Cases:

  • 'eras' returns '🌟'
  • 'ttpd' returns the emoji for "The Tortured Poets Dept." (🪶)
  • If era not found, returns '❓'

Example:

const reputationEmoji = getEraEmoji('reputation');
// Returns: '🐍'

# applyEraTheme(eraNameOrId)

Updates CSS variables in the document based on the selected era.

Parameters:

  • eraNameOrId (string): The era name or ID

Effects:

  • Sets CSS variables:
    • --era-primary
    • --era-secondary
    • --era-font
  • Stores preference in localStorage

# initializeTheme()

Initializes the theme from saved preference or defaults.

Effects:

  • Retrieves saved era from localStorage
  • Applies the corresponding theme or default theme if none saved

# Implementation Details

The theme system works by processing the design specifications and creating a map for easy access:

// Create a map for easier access by era name or ID
const themeByEra = {};

// Process the design specs data
designSpecs.forEach(spec => {
  const normalizedKey = spec.Era.toLowerCase().replace(/\s+/g, '-');
  const eraId = normalizedKey.replace(/\(|\)/g, '');
  
  themeByEra[spec.Era] = spec;
  themeByEra[normalizedKey] = spec;
  themeByEra[eraId] = spec;
});

This allows the system to look up era specifications by various forms of the era name.

# Color Utility Functions

The theme system includes a utility function to determine if a color is light or dark, which helps in selecting appropriate text colors:

function isLightColor(hexColor) {
  // Remove the # if it exists
  hexColor = hexColor.replace('#', '');
  
  // Convert to RGB
  const r = parseInt(hexColor.substr(0, 2), 16);
  const g = parseInt(hexColor.substr(2, 2), 16);
  const b = parseInt(hexColor.substr(4, 2), 16);
  
  // Calculate brightness (using YIQ formula)
  const brightness = (r * 299 + g * 587 + b * 114) / 1000;
  
  // Return true if the color is light
  return brightness > 155;
}

This is used internally by the getEraComponentStyle function to determine text colors for buttons.

# Usage in Components

To apply era-specific styling in components:

import { getEraTheme, getEraComponentStyle, getEraEmoji } from '@/utils/themeUtils';

export default {
  props: {
    era: {
      type: String,
      default: 'default'
    }
  },
  computed: {
    themeStyle() {
      return getEraComponentStyle(this.era, 'card');
    },
    eraEmoji() {
      return getEraEmoji(this.era);
    }
  }
}

# CSS Implementation

The theme system sets CSS variables that can be used throughout the application:

:root {
  --era-primary: #008080;    /* Default primary color */
  --era-secondary: #B2DFDB;  /* Default secondary color */
  --era-font: 'Outfit', sans-serif;  /* Default font */
}

These variables are used in component styles:

.era-card {
    border-color: var(--era-primary);
    border-width: 1px;
}

.era-active-tab {
    border-bottom-color: var(--era-primary) !important;
    color: var(--era-primary) !important;
}

.era-selected-item {
    background-color: var(--era-secondary);
    border: 1px solid var(--era-primary);
}

.era-button {
    background-color: var(--era-primary);
}

.era-rank-indicator {
    background-color: var(--era-primary);
}

.era-font {
    font-family: var(--era-font);
}

# User Preferences

The theme system stores the user's preferred era in localStorage:

// Store preference if needed
localStorage.setItem('preferred-era', eraNameOrId);

This preference is used to initialize the theme when the application loads:

export function initializeTheme() {
  const savedEra = localStorage.getItem('preferred-era');
  applyEraTheme(savedEra || 'default');
}

# Integration with Tailwind CSS

The application uses Tailwind CSS with custom colors defined in the configuration:

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./src/**/*.{html,js,vue}",
    "./src/index.html",
  ],
  theme: {
    extend: {
      colors: {
        // Add default theme colors
        'app-primary': '#008080',
        'app-secondary': '#B2DFDB',
      },
    },
  },
  plugins: [],
}

These colors provide a fallback for components that don't use the dynamic theme system.
