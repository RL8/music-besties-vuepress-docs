# Components Overview

This section provides a big picture overview of all the components used in the app. Components are designed to work with the era-specific theming system documented in the Design System section.

# Component Hierarchy

The application is structured around these key components, which are implemented in Vue.js:

Figure: Component Hierarchy

Tree diagram showing the Vue.js component hierarchy from App.vue down to individual UI elements. This updated diagram reflects the complete application structure including the navigation system and supporting views.

# Primary Components

  • App.vue: The root component that manages routing and global state, including theme initialization
  • DashboardView.vue: Main entry point displaying the Era Cards grid with era-specific styling
  • EditRankingsView.vue: Interface for selecting and ranking items with themed elements
  • SidebarPanel.vue: Context-sensitive panel for viewing rankings and reviews with era-specific styling
  • NavigationSystem.vue: Global navigation system accessible from all views

# Supporting Components

  • EraCard.vue: Reusable component for era display on the dashboard, styled with era-specific colors and fonts
  • SelectionTabs.vue: Tab navigation for different ranking categories, with active tabs styled based on era theme
  • ItemChip.vue: Interactive component for selection functionality, using era-specific styling for selected state
  • RankingsList.vue: Component for displaying ordered rankings with era-themed indicators
  • ReviewForm.vue: Form component for ratings and comments, styled according to the current era theme

# Navigation Components

  • NavMenu.vue: Menu component with navigation links
  • AboutView.vue: View component for application information
  • ProfileView.vue: View component for user profile
  • SettingsView.vue: View component for user settings, including theme selection options
  • TermsView.vue: View component for legal information

# Theme Integration

All components are designed to work with the era-specific theming system documented in the Theme System section. Components use CSS variables (--era-primary, --era-secondary, --era-font) to apply consistent styling based on the selected era.

Key themed elements include:

  • Era-specific card styling with themed borders and backgrounds
  • Themed active tab indicators
  • Era-specific selection indicators
  • Themed buttons and interactive elements
  • Era-specific fonts for themed content
